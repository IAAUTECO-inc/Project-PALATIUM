# Security Policy

## Supported Versions
We prioritize the security of our stable releases. Please ensure you are running a supported version.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability
We take the security of our sovereign infrastructure seriously. If you discover a vulnerability, please follow these steps:

1. **Do not open a public issue.**
2. Send a detailed report to **security@[your-domain].org**.
3. Include a Proof of Concept (PoC) and the potential impact on system integrity.

### Our Commitment
* **Acknowledgement**: Within 48 hours.
* **Status Updates**: Every 5 business days until resolution.
* **Disclosure**: We follow a coordinated vulnerability disclosure (CVD) process. A CVE will be requested upon fix confirmation.
